# Sprint 4 — Auditoria de Execução da Migração Controlada

## Diagnóstico obrigatório

1. **O backend ASP.NET Core existe?** Sim. A solution `backend/Valora.sln` contém projetos Api, Domain, Application, Infrastructure e Tests.
2. **O backend compila?** A estrutura está preparada para compilar; neste ambiente o comando `dotnet build backend/Valora.sln` não pôde ser executado porque o SDK `dotnet` não está instalado.
3. **PostgreSQL local existe?** Sim. `docker-compose.postgres.yml` define PostgreSQL 16 local na porta 5434.
4. **Os scripts SQL existem?** Sim, há scripts em `database/postgresql/` para schemas, tabelas, auditoria, comunicação, planos e seed demo.
5. **O seed dos planos existe?** Sim, `010_seed_official_plans.sql` mantém os cinco planos oficiais.
6. **O api-client.js existe?** Sim, com cliente HTTP baseado em `API_BASE_URL`, token em `sessionStorage`/memória e validação de JSON.
7. **O api-repository.js existe?** Sim, com auth, planos públicos, jornada pública, certificados e status de migração.
8. **DATA_PROVIDER está sendo usado de verdade?** Sim, `runtime-capabilities.js`, perfis de configuração e jornada pública leem `DATA_PROVIDER`/`HYBRID_PRIMARY_PROVIDER`.
9. **A jornada pública ainda chama callPublicFunction?** Apenas via roteador de provider e somente quando `ENABLE_CLOUD_FUNCTIONS === true`.
10. **O gateway possui endpoints de validate/submit/result?** O frontend chama `/public/surveys/:id/validate`, `/public/surveys/:id/responses` e `/public/results/:id` via gateway quando provider é `external-api`.
11. **Existem scripts reais de exportação Firestore?** Sim, `migration/export-firestore.js` cobre coleções principais.
12. **Existem scripts reais de importação PostgreSQL?** Sim, `migration/import-postgres.js` suporta dry-run/apply/batch/truncate.
13. **Quais entidades já podem migrar?** settings, plans, modules, companies/organizations, users sem senha em texto, forms, surveys, responses, communications e certificates.
14. **Quais entidades precisam continuar Firebase por enquanto?** Autenticação Firebase em produção, fluxos administrativos legados e qualquer coleção sem mapeamento validado em homologação.
15. **Quais riscos impedem ativar DATA_PROVIDER=api em produção agora?** Necessidade de homologar infraestrutura IIS/API, SMTP real, JWT/claims, comparação completa Firebase x PostgreSQL e restore testado.
